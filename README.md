# The Empathetic Conservator

Vision-based adaptive UR3 cobot for art restoration ergonomics.
ASM2 2026, Group 2 (TU Wien).

## Quick start

```bash
python -m venv .venv
source .venv/bin/activate          # Windows: .venv\Scripts\activate
pip install -r requirements.txt
```

## Project structure

```
ec/
├── src/
│   ├── config.py               # ALL tunable parameters
│   ├── pose_detector.py        # MediaPipe Pose wrapper
│   ├── pss_calculator.py       # Postural Strain Score
│   ├── intervention_policy.py  # when/how to intervene
│   ├── ur3_controller.py       # UR3 control via ur_rtde
│   ├── session_logger.py       # CSV logging
│   ├── main.py                 # session runner
│   └── analyze_results.py      # statistics + plots
├── scripts/
│   ├── test_ur3_connection.py  # Day 1 connection test
│   ├── nasa_tlx_form.py        # post-session questionnaire
│   └── score_pss_validation.py # PSS vs RULA validation
├── data/
│   ├── sessions/               # per-session CSV logs
│   ├── nasa_tlx.csv            # questionnaire results
│   ├── rula_scores.csv         # expert RULA scores for 20 photos
│   └── validation_photos/      # 20 standardized posture images
├── results/                    # generated by analyze_results.py
└── docs/                       # implementation plan, etc.
```

## Workflow

```bash
# Day 1: connect to robot
python scripts/test_ur3_connection.py

# Day 2-3: practice with simulator
python src/main.py --participant TEST --condition control --simulate --duration 2

# Day 4: pilot test (one team member as participant)
python src/main.py --participant PILOT --condition experimental --duration 10

# Day 5-6: real sessions
python src/main.py --participant P01 --condition control
python scripts/nasa_tlx_form.py
# break...
python src/main.py --participant P01 --condition experimental
python scripts/nasa_tlx_form.py

# After all sessions
python scripts/score_pss_validation.py  # PSS vs RULA on 20 photos
python src/analyze_results.py           # full statistical report
```

## Important notes

- The proposal mentions a "ROS interface" - we use **ur_rtde** instead because
  Pilotfabrik provides Polyscope only. Functionality is identical.
- N target = 6 (down from 6-8 in proposal) given realistic time budget.
- Sessions stay at 45 min as in proposal; use --duration to override.
